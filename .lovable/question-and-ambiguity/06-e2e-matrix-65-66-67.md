# 06 — E2E test matrix for scripts 65/66/67 on real Ubuntu and macOS

**Spec reference:** "Add and run an end-to-end test matrix for scripts
65/66/67 on real Ubuntu and macOS, including dry-run coverage and
root-requirement checks."

## Point of confusion

The request says **"on real Ubuntu and macOS"**, but this Lovable
sandbox is a single Linux container (no Mac runner attached, no GitHub
Actions matrix configured). The harness needs a sensible interpretation
of "real" coverage that:

1. Actually exercises both OS code paths today, on whatever box runs it.
2. Will light up genuine Mac coverage automatically the moment a Mac
   runner picks the same script up — no rewrite required.
3. Doesn't lie: cells that can't run on the current host must be
   reported as `SKIP` with a reason, not silently `PASS`.

## Options considered

- **Option A — CI matrix only.** Add a GitHub Actions workflow with
  `runs-on: [ubuntu-latest, macos-latest]` and call it done. _Pro:_
  textbook "real OS" coverage. _Con:_ requires CI infra the project may
  not have wired; nothing runs locally; no result for today's task.
- **Option B — Single host-aware matrix script (chosen).** One bash
  harness at `scripts-linux/_shared/tests/e2e/run-matrix.sh` that
  detects `uname`/`id -u` and runs every cell that's safe on the
  current host, skipping the rest with a clear reason. Per-folder smoke
  tests already use `uname` stubs to fake Darwin, so 66 is exercised
  end-to-end even on Linux; 65 runs natively on both; 67 is Linux-only.
  Same script can be invoked unchanged from a Mac runner later — the
  Darwin-only cells will simply flip from SKIP to PASS.
- **Option C — Container-based Mac emulation.** Spin up a Darwin
  toolchain via `osxcross` or similar to fake macOS on Linux. _Pro:_
  closer to "real" macOS. _Con:_ heavy, fragile, off-scope for a
  cleanup script harness.

## Recommendation / inference used

**Option B.** Created `scripts-linux/_shared/tests/e2e/run-matrix.sh`
with four cells:

1. Per-folder smoke tests on the current OS (always runs all three;
   66's smoke stubs `uname=Darwin` so it works on Linux CI).
2. Production-entrypoint dry-runs through `run.sh --dry-run` against a
   `mktemp` sandbox HOME — covers the operator path with no mutations.
3. OS-guard checks: 66 must refuse on Linux, 67 must refuse on macOS.
4. Root-requirement contract: `--scope user` must work for any UID;
   `--scope system` from a non-root UID must refuse or warn (never
   silently proceed to system writes).

Wired it as `bash scripts-linux/run.sh e2e-matrix` (aliases: `e2e`,
`test-matrix`) and added a help entry. Verified locally: **PASS=11,
FAIL=0, SKIP=4** (skips are the 4 Darwin-only / non-root-only cells
that this Linux+root sandbox can't honestly cover).

## How to revert

Remove `scripts-linux/_shared/tests/e2e/run-matrix.sh`, the
`e2e-matrix|e2e|test-matrix` case in `scripts-linux/run.sh`, the help
block entry for `e2e-matrix`, and the dispatch case for `e2e-matrix)`
near the bottom of `scripts-linux/run.sh`.
